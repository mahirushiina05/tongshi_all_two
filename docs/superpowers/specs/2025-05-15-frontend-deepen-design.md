# 前端页面深化设计方案

> AI 通识课平台 "学·练·创·行" 前端全面深化
> 日期：2025-05-15

## 背景

平台前端已实现 4 个模块的基础页面（学/练/创/行），但存在以下问题：
- 品牌仍显示 3 字 "探·创·行"，缺少 "练"
- 多处按钮无跳转目标（查看详情、上传作品、打开成长档案）
- 缺少做题交互、作品详情、成长档案等子页面
- 无信息页面（关于、隐私、联系）和 404 页面
- Footer 链接指向不存在的锚点

## 推进策略

分 3 批交付，每批可独立验证。

## 第 2 批：模块页面深化

### 2A. 做题交互页

**新文件：** `views/PracticeQuizView.vue`
**路由：** `/practice/quiz/:chapterId`

#### 页面结构

```
┌─────────────────────────────────────────┐
│ ← 退出练习    第 01 章 · 人工智能概述    │
│                 3 / 20 题               │
│  ████████░░░░░░░░░░░░  15%             │
├─────────────────────────────────────────┤
│                                         │
│  Q3. 以下哪个不是机器学习的类型？        │
│                                         │
│  ○ A. 监督学习                          │
│  ○ B. 无监督学习                        │
│  ◉ C. 随机学习                          │
│  ○ D. 强化学习                          │
│                                         │
│  ┌─ 提交答案 ─┐                         │
│                                         │
│  ┌─────────────────────────────┐        │
│  │ ✗ 回答错误                  │        │
│  │ 正确答案：C                 │        │
│  │ 解析：机器学习主要分为...    │        │
│  └─────────────────────────────┘        │
│                                         │
│         ← 上一题    下一题 →             │
│                                         │
│  [1][2][3✓][4✗][5][6]...[20]  题号导航  │
└─────────────────────────────────────────┘
```

#### 交互逻辑

1. 进入页面加载当前章节题目数据（mock）
2. 选择题：点击选项高亮，点击"提交答案"后显示结果
3. 填空题：输入框 + 提交，提交后显示结果
4. 答对：绿色提示 + 解析；答错：红色提示 + 正确答案 + 解析
5. "上一题/下一题"按钮切换，已提交的题可以回看
6. 底部题号导航条，✓ 表示答对，✗ 表示答错，当前题高亮
7. 全部完成后显示本章得分汇总

#### 数据结构（mock）

```typescript
interface Question {
  id: number
  type: 'choice' | 'fill'
  chapter: number
  stem: string        // 题干
  options?: string[]  // 选择题选项
  answer: string      // 正确答案
  explanation: string // 解析
}
```

## 数据流

当前阶段全部使用 mock 数据（hardcoded 数组），后续接入后端 API 时替换即可。每个页面的数据结构已在各页面设计中定义。

## 技术约束

- Vue 3 Composition API + `<script setup>` + TypeScript
- Element Plus 组件库（表单、按钮、标签、进度条等）
- ECharts（仅成长档案页的雷达图）
- 响应式断点：768px / 640px / 1024px
- 颜色体系：使用 main.css 中定义的 CSS 变量
- 不出现公司名称、无商业化内容
