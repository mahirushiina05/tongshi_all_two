# 多数据库协作与对象存储改造 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 在保持现有 `FastAPI + MySQL` 主体不变的前提下，引入 `SeaweedFS(S3)` 作为新文件默认存储，建立统一文件元数据与存储抽象层，并兼容历史 `backend/uploads` 文件与现有前端接口。

**Architecture:** 首期继续以 MySQL 作为唯一事务主库，新增 `StoredFile` 元数据模型和 `StorageService` 抽象层，把“文件内容存储”从业务表 URL 中解耦。新上传文件走 S3 适配器，历史文件走本地适配器，教师批量下载改为“临时 ZIP 文件 + 流式响应”，避免整包 `BytesIO` 常驻内存。

**Tech Stack:** FastAPI、SQLAlchemy、MySQL、SQLite（测试内存库）、SeaweedFS S3 网关、Python 标准库 `zipfile`/`tempfile`、Vue 3、TypeScript、Vite、Element Plus

```python
from dataclasses import dataclass
from typing import BinaryIO, Protocol

@dataclass
class StoredObject:
    storage_provider: str
    bucket_name: str
    object_key: str
    stored_name: str
    content_type: str
    size_bytes: int

class StorageAdapter(Protocol):
    def save_bytes(self, *, content: bytes, object_key: str, content_type: str) -> StoredObject: ...
    def open_stream(self, *, bucket_name: str, object_key: str) -> BinaryIO: ...
    def exists(self, *, bucket_name: str, object_key: str) -> bool: ...
    def delete(self, *, bucket_name: str, object_key: str) -> None: ...
```
