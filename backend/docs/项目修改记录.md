## 第 1 轮：基础安全检查

### 修改时间

2026-05-18

### 修改目标

修复后端代码中的功能正确性问题 — 包括答题计数逻辑 bug、ORM 关系缺失导致的 N+1 查询、文件上传无校验、批量导入事务缺陷四项高优先级问题。

### 修改内容

1. **修复 `quiz_service.submit_answer` 计数逻辑**
   - 文件：`app/services/quiz_service.py`
   - 问题：`session` 配置了 `autoflush=False`，先 `db.add(attempt)` 再执行 `count()` 查询时，新增 attempt 对 count 不可见，导致 `questions_done` 计数不准确
   - 修复：在 count 查询前显式调用 `db.flush()`；`sp.questions_done` 改为直接从 count 结果赋值，而非 `+= 1`

2. **`QuizAttempt` 模型添加 `question` 关系，消除 N+1 查询**
   - 文件：`app/models/entities.py`、`app/services/quiz_service.py`
   - `QuizAttempt` 新增 `question = relationship("Question")`
   - `get_quiz_history` 使用 `joinedload(QuizAttempt.question)` 预加载，替代原先循环内逐条查询 Question 的 N+1 模式

3. **文件上传增加安全校验**
   - 文件：`app/core/upload_validation.py`（新建）、`app/api/v1/routes/upload_routes.py`、`question_routes.py`、`class_routes.py`
   - 通用上传接口：限制允许的文件类型（图片/文档/视频/压缩包）和 50MB 上限
   - Excel 导入接口：限制 `.xlsx` / `.xls` 和 10MB 上限
   - 所有上传接口在 `file.read()` 之后、写磁盘之前做校验，早期拦截非法文件

4. **批量导入事务保护修复**
   - 文件：`app/services/class_service.py`
   - 问题：Excel 导入学生在循环中 catch `SQLAlchemyError` 后调用 `db.rollback()`，导致已成功处理的行也被回滚
   - 修复：每行用 `db.begin_nested()` 保存点包裹，单行失败仅回滚该行，其他行不受影响

### 验证情况

- 所有修改文件通过 Python 编译检查
- `upload_validation` 校验逻辑通过单元测试（正常文件、不支持类型、超限文件、空文件名）

### 遗留问题

无
